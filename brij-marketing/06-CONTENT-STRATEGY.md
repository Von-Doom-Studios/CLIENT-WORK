# Content Strategy — BRIJ

---

## Content Pillars

Every piece of content falls into one of four pillars. If it doesn't fit, it doesn't get made.

### Pillar 1: Proof It Works
**What:** Live demos, real bookings, actual transactions, user stories.
**Why:** BRIJ is asking people to believe something new. Proof is the only marketing that matters early.
**Formats:** Demo videos, screen recordings, code walkthroughs, transaction receipts, user-generated content.
**Frequency:** 2–3x/week

### Pillar 2: The Problem We Solve
**What:** Content that makes the audience feel the pain of the current system before showing the solution.
**Why:** People adopt new tools because the old way is broken, not because the new way is cool.
**Formats:** Before/after comparisons, "why flight booking is broken" threads, price manipulation exposés.
**Frequency:** 1–2x/week

### Pillar 3: How It Works (Under the Hood)
**What:** Technical content that builds credibility with developers.
**Why:** The geek audience needs to understand the architecture to trust it.
**Formats:** X threads, blog posts, GitHub docs, technical YouTube, conference talks.
**Frequency:** 1x/week

### Pillar 4: The Bigger Picture
**What:** Thought leadership on the agent economy, autonomous software, the future of commerce.
**Why:** Positions BRIJ as the company that defined the category.
**Formats:** Long-form threads, blog posts, podcast appearances.
**Frequency:** 1x/every 2 weeks (earn the right first — don't lead with vision)

## Content Formats

### Short-Form Video (15–60 seconds)
- **Purpose:** Discovery. Stop the scroll.
- Show the result in the first 3 seconds (a confirmed booking, a completed transaction)
- No setup footage (installing MCP, configuring API keys — that's a separate tutorial)
- Human-readable: numbered options (Flight 1, Flight 2), not reference IDs
- Maximum 5 words per text frame
- Use high-contrast hooks in the opening (see [references/social-media-frameworks.md](./references/social-media-frameworks.md) — Retention-Spike framework)
- **Channels:** X, TikTok, Instagram Reels, YouTube Shorts

### Technical Thread (X/Twitter)
- **Purpose:** Credibility. Shareable technical narrative.
- 5–10 tweets per thread
- First tweet is the hook — one compelling claim
- Include at least one code snippet
- End with a CTA (try it, read the spec, check the repo)
- No marketing language — just explain what it does and why
- **Channel:** X

### Demo/Tutorial Video (2–5 minutes)
- **Purpose:** Convert interest into usage.
- Start with the end result, then show how to get there
- Use best available model (Grok 4.3 beta or specified)
- Specify real airport codes, not city names
- Cut any section over 30 seconds without visual change
- **Channels:** YouTube, embedded on docs/landing pages

### Email Content
- **Purpose:** Nurture signups, announce launches, drive repeat usage.
- See [07-CHANNELS.md](./07-CHANNELS.md) — Email section for full strategy
- One message, one CTA per email
- Only send when there's real news
- **Channel:** Email

### Blog Post
- **Purpose:** SEO, credibility, reference material.
- One idea per post
- 800–1,500 words for technical. 500–800 for announcements.
- Working code examples where relevant
- Publish on BRIJ blog + dev.to simultaneously
- **Channels:** Blog, dev.to, cross-posted to X

## Weekly Production Workflow

### Thursday/Friday: Plan
Anthony drafts next week's content plan:
- What content? (format, topic, pillar)
- Which channel?
- Who creates it?
- What's the narrative thread?

### Monday: Execute anchor content
The biggest piece of the week goes live Monday (launch thread, demo video, etc.)

### Daily: Minimum one action
At least one piece of content or engagement action per day. Can be:
- A new post
- A repost with commentary
- Engaging with user content
- Replying to relevant threads

### Quality Gate
Before any content ships:

**Phase 1:** "Would this make a developer stop scrolling and think 'this is real'?"
**Phase 2+:** "Would this make a non-technical person say 'I want to try this'?"

If yes → ship. If no → revise or kill.

---

## Content Progression

The narrative builds across phases:

**Weeks 1–2:** "We built something new. Here's proof it works."
→ Demos, code, technical threads, builder reactions

**Weeks 3–6:** "Now anyone can use it. Here's what it feels like."
→ Consumer demos, comparison content, user stories

**Weeks 7+:** "This is how people book flights now."
→ Testimonials, savings calculators, press coverage, social proof

Each phase's content makes the next phase more credible.
